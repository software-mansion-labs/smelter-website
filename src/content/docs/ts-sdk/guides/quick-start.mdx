In this guide, you'll learn how to create a basic smelter setup using Smelter TypeScript SDK. We will show you how to take two input videos and produce an output video with both of the inputs displayed side-by-side.

<Steps>

1. Create Smelter object and initialize it.
   <Code code={`
     import Smelter from "@swmansion/smelter-node"

     async function start() {
       const smelter = new Smelter();
       await smelter.init();
     }
   `} lang='tsx' />

   It will download appropriate Smelter binaries and start the server on the local
   machine.

   <CollapsibleDetails summaryTitle='Alternatively, you can connect to an existing instance.'>
   <Code code={`
     import Smelter, { ExistingInstanceManager } from "@swmansion/smelter-node"

     async function start() {
       const manager = new ExistingInstanceManager({
         ip: "127.0.0.1",
         port: 8000,
         protocol: 'https',
       });
       const smelter = new Smelter(manager);
       await smelter.init();
     }
   `} lang='tsx' />
   </CollapsibleDetails>

2. Register an output stream and start the Smelter.

   <Code code={`
     import Smelter from "@swmansion/smelter-node"

     async function start() {
       const smelter = new Smelter();
       await smelter.init();

       await smelter.registerOutput("output", <View />, {
         type: "rtp_stream",
         transportProtocol: "udp",
         port: 9004,
         ip: "127.0.0.1",
         video: {
           resolution: { width: 1280, height: 720 },
           encoder": {
             type: "ffmpeg_h264",
             preset: "ultrafast"
           },
         },
         audio: {
           encoder: {
             type: "opus",
             channels: "stereo"
           },
         }
       });

       await smelter.start();
     }
   `} lang='tsx' collapse={["4-6"]} />

   This example creates an output that renders just an empty `<View />`. Resulting
   blank image is encoded and sent over UDP to local port 9004 using RTP protocol.

   <CollapsibleDetails summaryTitle='How to receive and display this stream?'>
     Above example sends that over UDP, so you need to start listening on that port before
     registering the output.

     #### GStreamer
     To view the output stream with GStreamer run the following command:
     <Code code={`
       gst-launch-1.0 rtpptdemux name=demux \\
         udpsrc port=9003 ! \"application/x-rtp\" ! queue ! demux. \\
         demux.src_96 ! \"application/x-rtp,media=video,clock-rate=90000,encoding-name=H264\" ! queue \\
         ! rtph264depay ! decodebin ! videoconvert ! autovideosink \\
         demux.src_97 ! \"application/x-rtp,media=audio,clock-rate=48000,encoding-name=OPUS\" ! queue \\
         ! rtpopusdepay ! decodebin ! audioconvert ! autoaudiosink sync=false
     `} lang='sh' frame='none' />  

     #### FFmpeg (`ffplay`)
     To view the output stream with FFmpeg:
     - Create the SDP file.
       <Code title="output.sdp" code={`
         v=0
         o=- 0 0 IN IP4 127.0.0.1
         s=No Name
         c=IN IP4 127.0.0.1
         m=video 9004 RTP/AVP 96
         a=rtpmap:96 H264/90000
         a=fmtp:96 packetization-mode=1
         a=rtcp-mux
       `} />
     - Run the following command
       <Code code={`ffplay -protocol_whitelist "file,rtp,udp" output.sdp`} lang="sh" frame="none"/>
     Note: FFmpeg does not support multiplexing audio and video on the same port, so this example
     will only work with video.
   </CollapsibleDetails>

   Or checkout other alternative protocols:
   <CollapsibleDetails summaryTitle='Option 1: WHIP output'>
   With WHIP output you can send it e.g. to Twitch.
   <Code code={`
     import Smelter from "@swmansion/smelter-node"

     async function start() {
       const smelter = new Smelter();
       await smelter.init();

       await smelter.registerOutput("output", <View />, {
         type: "whip",
         endpointUrl: "https://g.webrtc.live-video.net:4443/v2/offer",
         bearerToken: "<YOUR STREAM TOKEN>",
         video: {
           resolution: { width: 1280, height: 720 },
           encoder": { type: "ffmpeg_h264" },
         },
         audio: {
           encoder: { type: "opus", channels: "stereo" },
         }
       });

       await smelter.start();
     }
   `} lang='tsx' />
   </CollapsibleDetails>

   <CollapsibleDetails summaryTitle='Option 2: MP4 output'>
   With mp4 output you can record the output to file.
   <Code code={`
     import Smelter from "@swmansion/smelter-node"

     async function start() {
       const smelter = new Smelter();
       await smelter.init();

       await smelter.registerOutput("output", <View />, {
         type: "mp4",
         serverPath: "output.mp4",
         video: {
           resolution: { width: 1280, height: 720 },
           encoder: { type: "ffmpeg_h264" },
         },
         audio: {
           encoder: { type: "opus", channels: "stereo" },
         }
       });

       await smelter.start();
     }
   `} lang='tsx' />

   Note that mp4 must be gracefully terminated to write metadata,
   so if you just kill the application the file will be corrupted.
   When you want to end run
   ```tsx
   await smelter.unregisterOutput('output');
   ```
   </CollapsibleDetails>


3. Register 2 input streams.

    <Code code={`
    import Smelter from "@swmansion/smelter-node"
    import { View } from "@swmansion/smelter"

    async function start() {
        const smelter = new Smelter();
        await smelter.init();

        await smelter.registerInput("input_1", {
          type: "rtp_stream",
          transportProtocol: "tcp_server",
          port: 9001,
          video: {
            decoder: "ffmpeg_h264"
          },
          audio: {
            decoder: "opus"
          }
        })

        await smelter.registerInput("input_2", {
          type: "rtp_stream",
          transportProtocol: "tcp_server",
          port: 9002,
          video: {
            decoder: "ffmpeg_h264"
          },
          audio: {
            decoder: "opus"
          }
        })

        await smelter.registerOutput("output_1", <View /> {
          type: "rtp_stream",
          transportProtocol: "tcp_server",
          port: 9003,
          video: {
            resolution: { width: 1280, height: 720 },
            encoder": {
              type: "ffmpeg_h264",
              preset: "ultrafast"
            },
          },
          audio: {
            encoder: {
              type: "opus",
              channels: "stereo"
            },
          }
        })
    }
    `} lang='tsx'  collapse={["4-30"]} />

4. Render a single input stream and wire up the `App` component to the registered output.
    The `App` component will serve as the root of our component tree.   

    ```tsx ins={3-5, 35} collapse={8-34}
    import Smelter from "@swmansion/smelter-node"

    function App() {
      return <InputStream inputId="input_1" volume={0.9} />
    }

    async function start() {
        const smelter = new Smelter();
        await smelter.init();

        await smelter.registerInput("input_1", {
          type: "rtp_stream",
          transportProtocol: "tcp_server",
          port: 9001,
          video: {
            decoder: "ffmpeg_h264"
          },
          audio: {
            decoder: "opus"
          }
        })

        await smelter.registerInput("input_2", {
          type: "rtp_stream",
          transportProtocol: "tcp_server",
          port: 9002,
          video: {
            decoder: "ffmpeg_h264"
          },
          audio: {
            decoder: "opus"
          }
        })
        
        await smelter.registerOutput("output_1", <App />, {
          type: "rtp_stream",
          transportProtocol: "tcp_server",
          port: 9003,
          video: {
            resolution: { width: 1280, height: 720 },
            encoder": {
              type: "ffmpeg_h264",
              preset: "ultrafast"
            },
          },
          audio: {
            encoder: {
              type: "opus",
              channels: "stereo"
            },
          }
        })
    }
    ```

5. Modify the components to display both inputs inside of [`Tiles`](/ts-sdk/components/tiles).
    
    [`Tiles`](/ts-sdk/components/tiles) will handle transitions and scaling of the inputs automatically. 

    ```tsx collapse={11-59}
    import Smelter from "@swmansion/smelter-node"

    function App() {
      return (
        <Tiles backgroundColor="#4d4d4d">
            <InputStream inputId="input_1" volume={0.9} />
            <InputStream inputId="input_2" />
        </Tiles>
      )
    }

    async function start() {
        const smelter = new Smelter();
        await smelter.init();

        await smelter.registerInput("input_1", {
          type: "rtp_stream",
          transportProtocol: "tcp_server",
          port: 9001,
          video: {
            decoder: "ffmpeg_h264"
          },
          audio: {
            decoder: "opus"
          }
        })

        await smelter.registerInput("input_2", {
          type: "rtp_stream",
          transportProtocol: "tcp_server",
          port: 9002,
          video: {
            decoder: "ffmpeg_h264"
          },
          audio: {
            decoder: "opus"
          }
        })
        
        await smelter.registerOutput("output_1", <App />, {
          type: "rtp_stream",
          transportProtocol: "tcp_server",
          port: 9003,
          video: {
            resolution: { width: 1280, height: 720 },
            encoder": {
              type: "ffmpeg_h264",
              preset: "ultrafast"
            },
          },
          audio: {
            encoder: {
              type: "opus",
              channels: "stereo"
            },
          }
        })
    }
    ```
